# Part 05: 上下文視窗與「超憶症」比喻

接下來我們來聊聊 Context Window 的限制
雖然現在有些模型號稱有 100 萬、200 萬個 Token 的容量
但這不代表你可以無限制地塞東西進去
因為這會帶來兩個嚴重的問題：
1. **算力成本與速度**：Token 越多，運算越慢，錢也燒得越快
2. **Lost in the Middle**：研究發現，當資訊放在中間時，模型很容易忽略它

這裡李老師用了一個非常有意思的醫學比喻：**「超憶症 (Hyperthymesia)」**
超憶症患者能記住生命中每一天的所有細節
你可能會覺得這很棒，但實際上這是一種負擔
因為這些人往往難以進行抽象思考
他們的大腦被過多無意義的雜訊（比如十年前的早餐吃了什麼）塞滿了
導致在做決策時，無法從浩如煙海的細節中抽離出來

對 AI Agent 也是一樣
如果你讓它記住每一個瑣碎的步驟，它會變得很笨
它會陷入冗長的回憶中，而無法針對當前情境做出判斷
這就是為什麼我們需要 Context Engineering
去人為地製造「遺忘」和「篩選」
這不是在削減模型的能力，而是在優化它的智力
這也解釋了為什麼「資訊管理」在 Agent 時代變得比「訓練模型」更重要
