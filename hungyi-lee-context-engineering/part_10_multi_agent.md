# Part 10: Multi-Agent 與 Context 隔離

隨著任務複雜度的提升，單一 Agent 的負擔會越來越重
它要同時記住高層次的 Goal
又要處理底層的 Tool Log
還要應付各種報錯
這會導致 Context 變得非常混亂，甚至出現「精神分裂」
為了解決這個問題，李老師提出了一個核心策略：**Context 隔離**

具體做法是將大 Agent 拆分成多個小 Agent：
*   **Planner (總招)**：只負責規劃和任務分配，它的 Context 非常乾淨，只包含大目標和進度
*   **Executor (執行者)**：負責具體操作，它的 Context 包含詳細的 Tool Output
*   **Evaluator (評估者)**：負責檢查結果，它的 Context 包含驗證標準

透過這種「分工」，每個 Agent 只需要處理自己那一小塊 Context
資訊在傳遞時經過了「人為的過濾與摘要」
主模型不再需要看到所有的髒活細節
這種「隔離」不僅解決了長度限制問題
更提升了系統的穩定性
這也解釋了為什麼現在的 AI 應用越來越傾向於用一群模型
而不是一個超級模型來解決所有問題
管理這群模型的資訊流，就是 Context Engineering 的高級課題
