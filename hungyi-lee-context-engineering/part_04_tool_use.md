# Part 04: 工具呼叫與 Function Calling

AI Agent 另一個強大的地方在於它會使用工具
這在技術上我們稱為 **Function Calling**
對 LLM 來說，使用工具其實就是輸出一段特定格式的文字
比如它輸出一段 JSON，告訴系統它想用 `GoogleSearch` 這個 Function
外部系統執行完後，再把結果 (Observation) 餵回給模型

在這裡，李老師提到了一個非常有趣的觀察：
模型其實是有自己的「判斷力」的
當外部工具提供的資訊與模型內部的知識發生衝突時，模型並不會盲目相信工具
它會去比較、去推理
例如，如果搜尋結果說今天是 2026 年，而模型內部知識停在 2024 年
模型通常會意識到「外部資訊是更新的」，進而採納工具的建議
但如果工具回傳的資訊過於荒謬，模型也可能會選擇不相信

這告訴我們，Context Engineering 的另一個任務
就是提供足夠的「證據」和「邏輯鏈」
讓模型能在衝突的資訊中做出正確的判斷
這不再只是簡單的文字拼接
而是在設計一套資訊互動的規範
讓模型能在多變的環境下依然保持理智
