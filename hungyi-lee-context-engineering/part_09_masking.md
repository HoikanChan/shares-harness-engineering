# Part 09: 觀察遮蓋 (Observation Masking)

當 AI Agent 在進行軟體工程或處理大量數據時
它會呼叫工具並得到非常長的輸出，我們稱為 **Observation (觀察)**
研究顯示，在沒有優化的情況下，這些 Observation 佔據了約 84% 的 Token 消耗
特別是在 Read Code (讀取程式碼) 的任務中
有些程式碼幾千行，一次 Read 就會塞滿 Context

為了解決這個問題，工程師們發展出了 **Observation Masking (觀察遮蓋)** 技術
它的核心邏輯是：
1. 模型不需要每次都看到完整的工具輸出
2. 我們可以用簡單的佔位符 (Placeholder) 或總結來取代長輸出
3. 只有當模型明確要求「我想看某一段細節」時，才把該內容抓進 Context

這就像是給模型看一份「目錄」而不是整本書
模型可以先瀏覽目錄，決定去讀哪一章
這極大地節省了 Token 成本，也避免了模型被大量無關的程式碼干擾
分析發現，透過這種精細的遮蓋與篩選
我們可以用極少的 Token 就讓模型完成原本需要巨大 Context 才能做到的事
這也是 Context Engineering 中最實用的「省錢」招數
