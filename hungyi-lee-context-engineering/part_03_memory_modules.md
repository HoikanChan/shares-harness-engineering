# Part 03: 記憶管理的核心模組

當 AI Agent 運作一段時間後，它產生的資訊會非常驚人
如果你把所有的 Observation 和 Action 都丟進去
你的 Context Window 很快就會被塞爆
所以我們需要一套記憶管理系統
李宏毅老師將其拆解為三個核心模組：

1. **Read (讀取模組)**：
   這很像我們現在常說的 RAG (檢索增強生成)
   Agent 不會把所有歷史紀錄都丟進去
   而是根據目前的狀況，從「長期記憶資料庫」中篩選出最相關的片段
   
2. **Write (寫入模組)**：
   這是一個「判斷」的過程
   AI 會問自己：「這件事值得記下來嗎？」
   它會過濾掉瑣碎的雜訊，只將重要的資訊存入長期記憶
   
3. **Reflection (反思模組)**：
   Agent 會定期對過去的經驗進行「總結」或「反思」
   它會把零碎的 Observation 轉化為高層次的的想法或計畫
   這不僅節省了空間，更提升了模型的決策品質

透過這三個模組，我們就能解決模型「資訊過載」的問題
讓它在有限的 Context 內，依然能根據精挑細選過的記憶來做出精準的決策
這就是為什麼一個好的 Agent 系統，其背後的 Context 設計是非常複雜的
