# 視頻字幕處理工作流 (Reusable Workflow)

此工作流用於將長篇視頻的 `.srt` 字幕文件拆分成多個邏輯章節，並剔除元數據（時間戳與序號），僅保留純文本內容以便閱讀或進一步處理。

---

## 🛠 工具準備

- **`split_srt.py`**: 一個 Python 腳本，負責根據配置進行拆分與清洗。

---

## 📋 處理步驟

### 1. 內容分析 (Manual Analysis)
- 閱讀原始 `.srt` 文件（或觀看視頻），確定章節的劃分點。
- 記錄每個章節在原始 `.srt` 文件中對應的**起始行號**與**結束行號**（以 1 為起始索引）。

### 2. 編寫大綱 (Documenting)
- 根據分析結果，編寫一個 `outline.md`。
- 包含每個章節的名稱、時間範圍、核心內容摘要。
- 建議包含指向未來生成文件的鏈接，方便導航。

### 3. 配置腳本 (Configuration)
- 打開 `split_srt.py` 文件。
- 修改 `INPUT_SRT` 變量為你的字幕文件名。
- 修改 `SPLIT_TASKS` 列表，填入對應的文件名與行號範圍：
  ```python
  SPLIT_TASKS = [
      {"name": "part_01.srt", "start": 1, "end": 100},
      {"name": "part_02.srt", "start": 101, "end": 200},
      # ... 更多章節
  ]
  ```

### 4. 執行拆分 (Execution)
- 在終端運行腳本：
  ```bash
  python split_srt.py
  ```
- 腳本會自動過濾掉：
  - 空行
  - 序號行 (如 `1`, `2`, `3`)
  - 時間戳行 (如 `00:00:00,000 --> 00:00:02,000`)

---

## 💡 注意事項

- **行號準確性**: 請確保 `end` 行號包含該章節的最後一個文本行及其對應的序號。
- **編碼**: 腳本默認使用 `utf-8` 編碼，若源文件編碼不同請自行調整。
- **文件後綴**: 雖然輸出文件擴展名仍為 `.srt`（應客戶要求），但內容已轉為純文本。若需要純文本，可將 `name` 改為 `.txt`。
